Geeklogの機能

Geeklogは、PHP/MySQL(Microsoft SQL Server、PostgreSQLも) をベースとした、ダイナミックなWebコンテンツを管理するためのアプリケーションです。 インストール直後から、ブログエンジン、言い換えれば、コメント・トラックバック・多彩なRSSフォーマット・スパム防御などのCMS(Contents Management System: コンテンツ管理システム)に必須の機能をすべてサポートしたCMSとして機能します。

ブログ/CMS機能

  • 複数の話題で記事の作成・公開予約・投稿・編集ができる
  • 公開の準備が整うまでは記事を書きかけ（ドラフト）として保存できる
  • 記事で使用するHTMLタグの管理(HTMLタグのホワイトリスト使用)
  • ふさわしくない語句の規制(不快な語句のフィルタリング)
  • 記事の中に画像や他のコンテンツを埋め込むことができる
  • 複数の作者が記事を投稿できる
  • ユーザーからの投稿をサポート
  • コメント、トラックバック、ピングバックをサポート
  • よく使われるRSSフォーマット(RSS/RDF/Atomなど)のインポート・エクスポートをサポート
  • スパム検出・削除
  • 個人用カレンダーをサポートしたしっかりしたカレンダー機能
  • OpenID、OAuth、他のリモート認証方法をサポート
  • オープンソースのプラグインによる機能拡張

管理者用機能

  • 簡単に自動でインストールできるインストーラー
  • 国際化とローカル化を完全にサポート
  • 使いやすい管理者用インターフェース
  • Geeklogの全機能を完全に設定できる管理者用インターフェース
  • カスタムのルック&フィールをサポートしたユーザー指向のテーマエンジン

セキュリティ機能

  • ユーザーアカウントとアクセス制御
  • Geeklogの全オブジェクトを制御する*nixスタイルのセキュリティモデル（プラグインでも使用可）
  • 記事・話題ごとにアクセス制御できる
  • パスワードはすべて暗号化されて保存（生のパスワードを保存しない）
  • 管理者のアクセスはすべてログに記録される
  • エラーはすべてログに記録される
  • フォーム送信とSQL呼び出しの際に発生するエラーはすべてログに記録される

開発者向けの機能

  • 開発者向けのAPIでプラグイン開発をサポート
  • 完全に設定可能なブロックシステム。ブロックの中でのPHP使用もサポート
  • 整然としたコードベース。PHPのオブジェクト指向機能を使用
  • デフォルトのデータベース(直接サポートしているMySQL、PostgreSQL、MS SQL)以外への移植を可能にする軽量のデータベース抽象化レイヤ

はじめに

GeeklogはもともとSecurity Geeks WebサイトのためにJason Whittenburgによって作成されました。2001年初頭に、Jasonは他のことに時間を使おうと決心し、プロジェクトをIowa Outdoors運営のためにGeeklogを使っていたTony Bibbsに引き渡しました。3年後、TonyはプロジェクトをDirk Haunに引き渡し、それ以降Dirk HaunがGeeklogの管理を行っています。

Geeklogは正真正銘のオープンソースソフトウェアであり、GNU GPLv2ライセンスでリリースされています。Geeklogの管理は容易なはずですが、それでもシステムのいくつかのコンポーネントにアクセスする必要があるでしょう。

このドキュメントはまだ発展の途上にあるため、意図せずして省略された箇所や、説明が過剰なし不足している箇所があるかもしれません。Apache(またはIIS)、SQLデータベース(MySQL、PostgreSQL、MS SQL)、PHPをある程度使った経験があることを前提としています。
